Que posso fazer com este ambiente
Este ambiente permite-lhe visualizar o conjunto de pronlemas, submeter soluções aos problemas, colocar perguntas ao júri, ver todas as submissões com os resultados da avaliação, bem como o estado actual da classificação do concurso.

Como deve o programa ler os dados de entrada? Como devem ser escritos os resultados do programa?
Os dados de entrada são passados ao standard input do processo e os resultados devem ser escritos no standard output.

Tentei submeter o programa e obtive a mensagem "Programa com extensão inválida"!
Tem de submeter o código fonte do seu programa. Uma submissão é aceite para avaliação apenas se a extensão do ficheiro corresponder a uma das linguagens aceites para o concurso.

Qual é o significado das mensagens de avaliação?
As mensagens de avaliação são as seguintes:

Accepted: O seu programa passou todos os testes com sucesso;
Compile Time Error: o sistema não conseguiu compilar o seu programa com sucesso; o compilador retornou um erro ou aviso (warning);
Runtime Error: o seu programa abortou, isto é, terminou prematuramente devido a um erro de execução;
Time Limit Exceeded: o seu programa não terminou dentro do do tempo limite de execução;
Wrong Answer: o seu programa correu um ou mais casos de teste sem erros de execução mas produziu um resultado que não era o esperado ;
Presentation Error: o resultado parece correcto mas não está apresentado no formato pedido; dado que nem sempre é fácil distinguir esta mensagem da de wrong answer, só é enviada em casos óbvios;
Invalid Submission: a sua submissão não está conforme as extensões esperadas; ou tentou submeter sem usar a interface web;
Output Limit Exceeded: o seu programa gera um resultado demasiado longo para este problema; os limites dependem dos casos de teste, mas são geralmente baixos (o limite é cerca de 100KB);
Program Size Exceeded: o seu programa é demasiado longo e não sera considerado (limite é cerca de 100KB);
Requires Reevaluation: Por alguma razão o seu programa tera de ser re-avaliado;
Invalid Function: O seu programa viola uma regra do concurso como chamar uma biblioteca não standard; também pode terminar com exit(1) ou mesmo return 1, uma uma instrução semelhante que indique um erro de execução.

Quando pressiono o botão de submissão não acontece nada!
Deve verificar a política de cache do seu navegador (browser). As páginas deveriam ser produzidas de cada vez que alguma actualização é detectada. Se as páginas não forem pedidas ao servidor nunca são actualizadas no seu navegador. No om the server, they will not be updated in your browser. In Mozilla e no Netscape verifique no menu "Edit" | "Preferences." | "Advanced" | "Cache" e escolha "Every time (I view the page)".

O formato da interface tem uma aparência estranha
Tente escolher um tipo de letra menor ("Edit" | "Preferences." | "Appearance" "Fonts" ). Alternativamente, tente remover barras de navegação ou pessoais no se navegador (menu "View").

Pressionei o botão "Submeter" e obtive a mensagem "Program não especificado".
Tem de seleccionar um ficheiro com o código fonte. Pressione "Browse..." para abrir um diálogo de selecção de ficheiro ou escreva o nome caminho na caixa à esquerda deste botão.
